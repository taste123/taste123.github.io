Materi ini membahas dasar-dasar API (Application Programming Interface) Testing, sebuah lapisan pengujian penting yang berfokus pada logika bisnis dan komunikasi data antar sistem.

## ğŸŒ Apa Itu API Testing?

> **API Testing** adalah proses pengujian yang dilakukan langsung pada *Application Programming Interface* (API) untuk memastikan bahwa API tersebut berfungsi sesuai dengan spesifikasi. Tujuannya adalah untuk memvalidasi bahwa API dapat menangani berbagai skenario, menerima *input* tertentu, dan menghasilkan *output* yang benar.

---

## ğŸ¯ Kenapa API Testing Penting?

Pengujian API memiliki beberapa keunggulan strategis dalam siklus pengembangan:

* **Memastikan Fungsionalitas:** Memverifikasi bahwa API berfungsi sesuai spesifikasi dan menghasilkan output yang benar.
* **Deteksi Bug Lebih Awal:** Meningkatkan keandalan sistem dengan menemukan bug pada lapisan logika bisnis sebelum diimplementasikan ke UI (User Interface).
* **Menjamin Keamanan:** Membantu mengidentifikasi celah keamanan dan melindungi API dari akses yang tidak sah.
* **Validasi Performa:** Mengukur stabilitas dan performa API di bawah beban tertentu (beban puncak).
* **Mempercepat Pengembangan:** Memungkinkan otomatisasi pengujian yang cepat, sehingga mempercepat siklus rilis.
* **Integrasi yang Lancar:** Menjadi pondasi penting untuk memastikan integrasi antar sistem atau layanan berjalan dengan baik.

---

## ğŸ” Anatomi Request & Response API

Komunikasi API terdiri dari dua bagian utama:

1.  **Request (Permintaan):** Pesan yang dikirim oleh klien ke server untuk mengakses atau memanipulasi data.
    * **Method (HTTP Verb):** Menentukan aksi yang ingin dilakukan (Contoh: `GET` untuk mengambil data, `POST` untuk mengirim data baru, `PUT` untuk memperbarui, `DELETE` untuk menghapus).
    * **URL (Endpoint):** Alamat unik dari *resource* (sumber daya) yang ingin diakses.

2.  **Response (Balasan):** Balasan yang diberikan oleh server setelah memproses *request*.
    * **Status Code:** Kode numerik yang menandakan hasil dari *request* (Contoh: `200 OK` - Sukses, `404 Not Found` - Tidak ditemukan, `401 Unauthorized` - Butuh otentikasi).

## ğŸ› ï¸ Tools API Testing Populer

Ada banyak *tools* yang digunakan untuk melakukan API testing, namun dua yang paling umum dibahas adalah:

### 1. Postman

Sebuah platform kolaborasi yang sangat populer untuk API *development* dan *testing*.

* **Fitur Utama:**
    * Mengirim request HTTP (GET, POST, PUT, dll).
    * Membuat *Collection* (kumpulan request).
    * *Testing* otomatis.
    * Pengelolaan *Environment* (misal: *dev*, *staging*, *prod*).
    * Membuat dokumentasi API dan *Mock server*.
* **Mengapa Populer:**
    * Sangat *User-Friendly* (mudah digunakan).
    * Manajemen koleksi yang mudah.
    * Dukungan *environment* dan variabel.
    * Visualisasi *response* yang baik.

### 2. SoapUI

Sebuah *tool* pengujian API *open-source* yang kuat, terutama untuk skenario yang lebih kompleks.

* **Fitur Utama & Kelebihan:**
    * Mendukung pengujian API **SOAP** (berbasis XML) dan **REST** (berbasis JSON/XML).
    * Bisa membuat *functional testing*, *security testing*, dan *load testing*.
    * Mendukung *data-driven testing* (mengimpor data dari Excel atau database).
    * Sangat cocok untuk *enterprise testing* dengan skenario yang kompleks.
