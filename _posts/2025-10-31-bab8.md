## ğŸŒ Apa Itu Cypress?

> **Cypress** adalah *framework* **end-to-end (E2E) testing** generasi baru yang dibuat untuk aplikasi web modern (seperti yang dibangun dengan React, Vue, Angular, dll).

### ğŸ”º Posisi Cypress dalam Piramida Pengujian

Meskipun Cypress bersinar di level **End-to-End (E2E) Testing**, ia juga sangat mampu digunakan untuk **Integration Testing** dan **Component Testing**, menjadikannya alat yang fleksibel.

---

## ğŸŒŸ Keunggulan Utama Cypress

Cypress memiliki beberapa keunggulan fundamental yang membedakannya dari *tools* automasi tradisional (seperti Selenium):

* **Test Runner Interaktif:** Ini adalah fitur andalan Cypress. Anda bisa melihat perintah dieksekusi secara visual dan *real-time*, serta melihat *snapshot* aplikasi pada setiap langkah.
* **Time Travel:** Memungkinkan Anda untuk "kembali ke masa lalu" dengan mengarahkan kursor ke log perintah di Test Runner untuk melihat *snapshot* DOM aplikasi pada saat itu, membuat *debugging* menjadi sangat cepat.
* **Automatic Waits:** Cypress secara otomatis menunggu elemen muncul di DOM dan menjadi *actionable* (misalnya, tidak terhalang, tidak *disabled*). Anda tidak perlu lagi menulis *explicit waits* atau *sleeps* secara manual.
* **Arsitektur Modern:** Cypress berjalan *langsung di dalam browser* bersama aplikasi Anda, memberinya kontrol penuh atas alur aplikasi dan lalu lintas jaringan tanpa perlu *driver* eksternal.

## ğŸ› ï¸ Setup dan Instalasi

Memulai Cypress sangatlah mudah dan cepat.

* **Kebutuhan:** Node.js terinstal di sistem Anda.
* **Perintah Instalasi:**
    1.  Inisialisasi proyek Node.js:
        `npm init -y`
    2.  Instal Cypress sebagai *dev dependency*:
        `npm install cypress --save-dev`
* **Membuka Test Runner:**
    `npx cypress open`

## âŒ¨ï¸ Perintah Dasar Cypress

Perintah Cypress sangat intuitif dan mudah dibaca, dirancang untuk meniru alur pikir pengguna.

| Perintah | Deskripsi |
| :--- | :--- |
| `cy.visit('URL')` | Membuka halaman web di *browser* |
| `cy.get('selector')` | Mengambil elemen DOM berdasarkan *selector* CSS |
| `cy.contains('text')` | Mengambil elemen berdasarkan teks di dalamnya |
| `cy.type('text')` | Mengetik ke dalam sebuah *input field* |
| `cy.click()` | Melakukan klik pada tombol, *link*, atau elemen lain |
| `cy.url()`` ` | Mendapatkan URL yang sedang aktif |
| `.should('assertion')`| Melakukan *assertion* (validasi), misal: `.should('be.visible')` |

## ğŸ Kesimpulan

Cypress adalah *framework testing end-to-end* modern yang kuat, dirancang untuk membuat proses pengujian menjadi lebih cepat, lebih mudah, dan lebih andal. Dengan fitur unggulan seperti Test Runner interaktif, *Time Travel*, dan *automatic waits*, Cypress secara efektif menyederhanakan *workflow* pengujian UI dan meningkatkan kualitas aplikasi web.
